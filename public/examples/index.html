How Does It Work?

Have an idea for a new product or feature and want to test if people will take action, like paying or signing up? Fake door testing is a proven method to validate your riskiest assumption—usually a user's willingness to pay or sign up. Payflow Insights lets you quickly create custom payment or sign-up flows to run these experiments. Here's how it works

1. The Set-Up

Provide a business name, product name, logo image, and product image. These fields are optional, so you can customize the experience as you like.

2. Select the Use Case

Choose from three options: one-time payment, subscription, or account creation. Then set the price to test for payment options.

3. Write a Thank You

If a user attempts to buy your product, they'll see a dialog explaining the experience isn’t ready. You can offer product updates if they opt in. Without opting in, only the transaction attempt is recorded—no personal information is stored. Payflow Insights never stores payment information under any circumstances and only collects personal data with user consent.

4. Generate Your Flow

Click “Generate Link” to get a hosted link, embeddable code, and a secret key to track analytics. DO NOT LOSE YOUR SECRET KEY!

5. Embed or Use the Link

Simply copy and paste the embed code to add the payment to any web page if you don’t want to send users to another domain. Alternatively you can use the link and prepend it by its self or prepend another link if you desire the user to go somewhere else after attempting to purchase.
